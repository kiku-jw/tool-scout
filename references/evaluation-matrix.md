# Evaluation Matrix

Use this structure when comparing candidates.

## Suggested columns

- tool
- type
- maturity
- why it fits
- main limitations
- pricing or cost shape
- integration path
- link

## Questions to answer

- Is it maintained now?
- Does it solve the actual job or just part of it?
- Is it cheap in engineering time, not just sticker price?
- Does it fit the current stack?
- Does it create vendor lock-in or review risk?

## Recommendation pattern

End with three short calls:

- `Try first`
- `Avoid for now`
- `Build ourselves instead` when appropriate
